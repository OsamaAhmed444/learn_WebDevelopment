أساسيات لغة HTML

HTML أو HyperText Markup Language هي اللغة الأساسية المستخدمة لبناء صفحات الويب. تُستخدم لتحديد هيكل الصفحة والعناصر المختلفة التي تظهر للمستخدم داخل المتصفح.

دور لغة HTML

تُعد HTML أساس بنية الويب، حيث يتم من خلالها إنشاء العناصر المختلفة التي تتكون منها صفحة الويب مثل العناوين والفقرات والصور والروابط والنماذج.

عناصر HTML

تُستخدم عناصر HTML لتمثيل المحتوى داخل الصفحة. تتكون أغلب العناصر من وسم فتح ووسم إغلاق يحتويان على المحتوى بينهما.

<h1>عنوان رئيسي</h1> <p>هذه فقرة نصية</p>

بنية مستند HTML

يتكون مستند HTML عادةً من قسمين رئيسيين: قسم head الذي يحتوي على البيانات الوصفية والروابط الخاصة بالملفات الخارجية، وقسم body الذي يحتوي على المحتوى الظاهر للمستخدم.

<html> <head> <title>My Page</title> </head> <body> Content Here </body> </html>

العناصر الشائعة في HTML

هناك العديد من العناصر المستخدمة بشكل متكرر أثناء بناء صفحات الويب مثل: العناوين من h1 إلى h6، والفقرات p، وحاويات div.

<h1>Main Heading</h1> <h2>Sub Heading</h2> <p>Paragraph Text</p> <div>Container</div>

عنصر div

يُعتبر العنصر div من أكثر العناصر استخدامًا في HTML. لا يحمل أي معنى دلالي خاص، ولكنه يُستخدم كحاوية عامة لتنظيم وتجميع العناصر الأخرى داخل الصفحة.

<div> <h2>Section Title</h2> <p>Section Content</p> </div>

العناصر الفارغة

بعض عناصر HTML لا تحتوي على وسم إغلاق لأنها لا تحتوي على محتوى داخلي، وتُعرف بالعناصر الفارغة.

<img src="image.jpg" alt="Image"> <br> <hr>

السمات Attributes

السمات هي معلومات إضافية تُضاف إلى عناصر HTML لتحديد سلوكها أو خصائصها. تُكتب السمات داخل وسم الفتح.

<a href="https://example.com"> Visit Website </a>
المعرفات والفئات

المعرفات ID

المعرف أو ID هو قيمة فريدة تُستخدم لتمييز عنصر واحد فقط داخل الصفحة. لا يجب تكرار نفس المعرف لأكثر من عنصر.

<div id="header"> Header Content </div>

الفئات Classes

تُستخدم الفئات لتجميع مجموعة من العناصر معًا بهدف تطبيق نفس التنسيق أو السلوك عليها. يمكن استخدام نفس الفئة مع أكثر من عنصر.

<p class="text">Paragraph One</p> <p class="text">Paragraph Two</p>
الأحرف الخاصة والربط

كيانات HTML

تُستخدم كيانات HTML لعرض بعض الأحرف الخاصة التي لا يمكن كتابتها مباشرة داخل الصفحة.

&amp; => & &lt; => < &gt; => >

العنصر link

يُستخدم العنصر link لربط ملفات CSS الخارجية بصفحة HTML.

<link rel="stylesheet" href="style.css">

العنصر script

يُستخدم العنصر script لربط ملفات JavaScript الخارجية أو كتابة أكواد JavaScript داخل الصفحة.

<script src="script.js"></script>
قالب HTML الأساسي

يُعتبر قالب HTML الأساسي نقطة البداية لأي صفحة ويب. يحتوي على العناصر الرئيسية اللازمة لعمل الصفحة بشكل صحيح داخل المتصفح.

بنية قالب HTML

يتكون القالب الأساسي من تعريف نوع المستند DOCTYPE والعناصر html و head و body. يتم وضع البيانات الوصفية داخل head بينما يوضع المحتوى المرئي داخل body.

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My Page</title> </head> <body> Content Here </body> </html>

أهمية قالب HTML

يساعد هذا القالب المتصفح على فهم نوع المستند وطريقة عرضه بشكل صحيح. ويُعد نقطة البداية القياسية لإنشاء أي صفحة ويب حديثة.

ترميز الأحرف UTF-8

يُستخدم ترميز UTF-8 لضمان عرض الأحرف والرموز المختلفة بشكل صحيح بغض النظر عن لغة المحتوى أو نظام التشغيل المستخدم.

إضافة ترميز UTF-8

يجب إضافة وسم meta الخاص بالترميز داخل عنصر head.

<meta charset="UTF-8">

أهمية UTF-8

يسمح بعرض النصوص العربية والإنجليزية واللغات الأخرى والرموز الخاصة بطريقة صحيحة داخل المتصفح.

تحسين محركات البحث والعلامات الوصفية

علامة Meta Description

تُستخدم العلامة الوصفية Description لتوفير وصف مختصر لمحتوى الصفحة. يظهر هذا الوصف غالبًا داخل نتائج البحث ويساعد في تحسين محركات البحث SEO.

<meta name="description" content="Learn HTML from beginner to advanced level." >

أهمية Meta Description

تساعد محركات البحث والمستخدمين على فهم محتوى الصفحة قبل زيارتها. كما يمكن أن تؤثر على معدل النقر على نتائج البحث.

علامات Open Graph

تُستخدم علامات Open Graph لتحسين طريقة ظهور صفحات الويب عند مشاركتها على مواقع التواصل الاجتماعي مثل Facebook و LinkedIn.

إضافة بيانات Open Graph

<meta property="og:title" content="Learn HTML"> <meta property="og:description" content="Complete HTML Tutorial"> <meta property="og:image" content="image.jpg">

فائدة Open Graph

تساعد في عرض عنوان ووصف وصورة مخصصة عند مشاركة الرابط على وسائل التواصل الاجتماعي.

عناصر الوسائط وتحسينها

العناصر المستبدلة

العناصر المستبدلة هي عناصر يتم استبدال محتواها بمصدر خارجي مثل الصور ومقاطع الفيديو والإطارات المضمنة.

<img src="photo.jpg" alt="Photo"> <iframe src="page.html"></iframe>

تحسين الوسائط

يُقصد بتحسين الوسائط تقليل حجم الصور والفيديوهات وتحسين أدائها لتسريع تحميل صفحات الويب.

تنسيقات الصور الشائعة

هناك عدة تنسيقات مستخدمة للصور ولكل منها استخدام مختلف.

JPG - للصور الفوتوغرافية PNG - للصور ذات الخلفية الشفافة GIF - للصور المتحركة WEBP - حجم أصغر وجودة أفضل

تراخيص الصور

يجب التأكد من حقوق استخدام الصور قبل إضافتها إلى موقع الويب. بعض الصور مجانية وبعضها يتطلب ترخيصًا أو إذنًا من صاحبها.

رسومات SVG

SVG هي رسومات متجهة قابلة للتوسيع يمكن تكبيرها أو تصغيرها دون فقدان الجودة. وتُستخدم بشكل واسع في الشعارات والأيقونات.

<svg width="100" height="100"> <circle cx="50" cy="50" r="40" /> </svg>
تكامل الوسائط المتعددة

عنصر الصوت audio

يُستخدم العنصر audio لتضمين ملفات صوتية داخل صفحات الويب.

<audio controls> <source src="audio.mp3"> </audio>

عنصر الفيديو video

يُستخدم العنصر video لتشغيل ملفات الفيديو مباشرة داخل صفحة الويب.

<video controls width="400"> <source src="video.mp4"> </video>

التضمين باستخدام iframe

يُستخدم العنصر iframe لتضمين محتوى خارجي داخل الصفحة مثل مقاطع YouTube أو الخرائط أو صفحات ويب أخرى.

<iframe src="https://www.youtube.com/embed/video-id"> </iframe>
الروابط والمسارات

تُستخدم الروابط في HTML للانتقال بين صفحات الويب أو أجزاء مختلفة من نفس الصفحة. ويتم إنشاؤها باستخدام العنصر a مع السمة href.

إنشاء رابط

يُستخدم العنصر a لإنشاء رابط قابل للنقر ينقل المستخدم إلى صفحة أو موقع آخر.

<a href="https://example.com"> Visit Website </a>

أنواع السمة target

تُستخدم السمة target للتحكم في مكان فتح الرابط عند النقر عليه.

<a href="https://example.com" target="_self"> Open In Same Tab </a> <a href="https://example.com" target="_blank"> Open In New Tab </a>

المسارات المطلقة Absolute Paths

المسار المطلق هو المسار الكامل الذي يحتوي على عنوان الموقع بالكامل. يُستخدم عادةً للربط بمواقع أو ملفات موجودة خارج المشروع.

<img src="https://example.com/images/photo.jpg">

المسارات النسبية Relative Paths

المسار النسبي يعتمد على موقع الملف الحالي داخل المشروع. ويُستخدم للوصول إلى الملفات الموجودة داخل مجلدات المشروع.

<img src="images/photo.jpg">

بناء جملة المسارات

توجد عدة رموز تُستخدم للتنقل بين الملفات والمجلدات داخل المشروع.

./ Current Directory ../ Parent Directory folder/file.html

حالات الروابط

تمتلك الروابط حالات مختلفة أثناء تفاعل المستخدم معها مثل الحالة الافتراضية وحالة المرور بالمؤشر وحالة الضغط.

a:link a:visited a:hover a:active

الروابط الداخلية

يمكن إنشاء روابط تنقل المستخدم إلى قسم معين داخل نفس الصفحة باستخدام المعرفات IDs.

<a href="#contact"> Go To Contact Section </a> <section id="contact"> Contact Information </section>
أهمية HTML الدلالية

العناصر الدلالية أو Semantic Elements هي عناصر تضيف معنى واضحًا لمحتوى الصفحة. وهي تساعد المطورين ومحركات البحث والتقنيات المساعدة على فهم بنية الموقع بشكل أفضل.

التسلسل الهرمي للعناوين

يجب استخدام عناصر العناوين بالترتيب الصحيح للحفاظ على هيكل منطقي للمحتوى. العنصر h1 يمثل أعلى مستوى بينما h6 يمثل أدنى مستوى.

<h1>Main Title</h1> <h2>Section Title</h2> <h3>Sub Section</h3>

العناصر التقديمية القديمة

كانت بعض العناصر تُستخدم للتحكم في مظهر المحتوى فقط دون إضافة معنى دلالي. وقد أصبحت مهملة في HTML الحديثة.

<center>Text</center> <font>Text</font> <big>Text</big>

العناصر الدلالية الحديثة

توفر العناصر الدلالية معنى واضحًا لأجزاء الصفحة المختلفة وتساعد على تنظيم المحتوى بطريقة احترافية.

العناصر الدلالية الأساسية

عنصر header

يمثل المحتوى التمهيدي للصفحة أو القسم مثل الشعار والعنوان الرئيسي.

<header> Website Header </header>

عنصر nav

يحتوي على روابط التنقل الرئيسية داخل الموقع.

<nav> Navigation Links </nav>

عنصر section

يُستخدم لتجميع مجموعة من المحتويات المرتبطة بموضوع واحد.

<section> Section Content </section>

عنصر article

يمثل محتوى مستقلاً يمكن فهمه بمفرده مثل المقالات أو المنشورات.

<article> Blog Post </article>

عنصر aside

يُستخدم للمحتوى الجانبي أو المعلومات المرتبطة بالمحتوى الرئيسي.

<aside> Sidebar Content </aside>

عنصر footer

يمثل تذييل الصفحة أو القسم ويحتوي غالبًا على معلومات التواصل وحقوق النشر.

<footer> Copyright 2026 </footer>

مثال على استخدام العناصر الدلالية

<header>Header</header> <nav>Navigation</nav> <main> <section> <article> Article Content </article> </section> <aside> Sidebar </aside> </main> <footer> Footer </footer>
العناصر الدلالية النصية

توفر HTML مجموعة من العناصر الدلالية المستخدمة لإعطاء معنى إضافي للنصوص داخل الصفحة. تساعد هذه العناصر المتصفحات ومحركات البحث والتقنيات المساعدة على فهم المحتوى بشكل أفضل.

عنصر التأكيد em

يُستخدم العنصر em لتمثيل النص الذي يحتاج إلى تأكيد أو تشديد أثناء القراءة. عادةً ما يظهر النص بخط مائل.

<p> I <em>really</em> like HTML. </p>

عنصر الأهمية القوية strong

يُستخدم العنصر strong للإشارة إلى أن النص يحمل أهمية كبيرة أو تحذيرًا أو معلومة مهمة.

<p> <strong>Warning:</strong> Save your work frequently. </p>

عنصر لفت الانتباه b

يُستخدم العنصر b لجذب الانتباه إلى نص معين دون إعطائه أهمية دلالية خاصة.

<b>New</b>

عنصر النص الاصطلاحي i

يُستخدم العنصر i لعرض النصوص ذات المعنى المختلف مثل المصطلحات الأجنبية أو التقنية أو الأفكار.

<i>Bonjour</i>

عنصر الكود code

يُستخدم العنصر code لتمثيل جزء من كود برمجي داخل النص.

<code> console.log("Hello"); </code>

عنصر الاقتباس المضمن q

يُستخدم العنصر q لعرض اقتباس قصير داخل النص.

<q> Knowledge is power. </q>

عنصر الاقتباس الكتلي blockquote

يُستخدم العنصر blockquote لعرض اقتباسات طويلة مأخوذة من مصدر آخر.

<blockquote> The future belongs to those who learn more skills. </blockquote>

عنصر الاختصار abbr

يُستخدم العنصر abbr لتمثيل الاختصارات مع توضيح معناها الكامل.

<abbr title="HyperText Markup Language"> HTML </abbr>

عنصر معلومات التواصل address

يُستخدم العنصر address لعرض معلومات التواصل الخاصة بشخص أو مؤسسة.

<address> contact@example.com </address>

عنصر الوقت time

يُستخدم العنصر time لتمثيل تاريخ أو وقت محدد بطريقة يمكن للمتصفحات فهمها.

<time datetime="2026-06-20"> June 20, 2026 </time>

عنصر الأس sup

يُستخدم العنصر sup لعرض النصوص المرتفعة مثل الأسس الرياضية.

x<sup>2</sup>

عنصر النص السفلي sub

يُستخدم العنصر sub لعرض النصوص المنخفضة مثل الصيغ الكيميائية.

H<sub>2</sub>O

عنصر التعليق غير المفصل u

يُستخدم العنصر u لتمييز جزء من النص يحتاج إلى ملاحظة أو تعليق.

<u> Important Note </u>

عنصر الشطب s

يُستخدم العنصر s لتمثيل المحتوى الذي لم يعد صحيحًا أو صالحًا.

<s>$100</s> $75

قائمة الوصف dl

تُستخدم قائمة الوصف لعرض المصطلحات وتعريفاتها بطريقة منظمة.

<dl> <dt>HTML</dt> <dd> Markup Language </dd> <dt>CSS</dt> <dd> Styling Language </dd> </dl>

عنصر Ruby Annotation

يُستخدم العنصر ruby لإضافة شروحات أو نطق إضافي لبعض الأحرف خاصة في اللغات الآسيوية.

<ruby> 漢 <rt>Kan</rt> </ruby>
عناصر وخصائص نماذج HTML

تُستخدم النماذج Forms لجمع البيانات من المستخدمين وإرسالها إلى الخادم لمعالجتها. وتُعد من أهم العناصر المستخدمة في مواقع التسجيل وتسجيل الدخول والتواصل والاستبيانات.

العنصر form

يُستخدم العنصر form لإنشاء نموذج يحتوي على حقول الإدخال المختلفة التي يتفاعل معها المستخدم.

<form action="submit.php" method="POST"> </form>

السمة action

تُستخدم السمة action لتحديد عنوان الصفحة أو الخادم الذي سيتم إرسال بيانات النموذج إليه بعد الضغط على زر الإرسال.

<form action="register.php"> </form>

السمة method

تُستخدم السمة method لتحديد طريقة إرسال البيانات. أشهر الطرق المستخدمة هي GET و POST.

<form method="POST"> </form>

مثال على نموذج كامل

<form method="POST" action="register.php"> <input type="text"> <input type="submit"> </form>
العنصر input

يُستخدم العنصر input لإنشاء حقول الإدخال المختلفة التي تسمح للمستخدم بإدخال البيانات أو اختيار الخيارات.

النوع text

يُستخدم لإنشاء حقل نصي عادي.

<input type="text">

النوع email

يُستخدم لإدخال البريد الإلكتروني مع التحقق من صحة التنسيق.

<input type="email">

النوع password

يُستخدم لإدخال كلمات المرور مع إخفاء الأحرف أثناء الكتابة.

<input type="password">

النوع number

يُستخدم لإدخال القيم الرقمية فقط.

<input type="number">

النوع radio

يُستخدم لاختيار خيار واحد فقط من مجموعة خيارات.

<input type="radio">

النوع checkbox

يُستخدم لاختيار أكثر من خيار من مجموعة خيارات.

<input type="checkbox">

النوع date

يُستخدم لاختيار تاريخ من تقويم مدمج في المتصفح.

<input type="date">
خصائص حقول الإدخال

السمة placeholder

تُستخدم لعرض نص إرشادي داخل حقل الإدخال لمساعدة المستخدم.

<input type="text" placeholder="Enter Your Name">

السمة value

تُستخدم لتحديد قيمة افتراضية لحقل الإدخال أو لتحديد النص الظاهر داخل الأزرار.

<input type="text" value="Osama">

السمة name

تُستخدم لتحديد اسم الحقل الذي سيتم إرساله مع البيانات. وتُعتبر المفتاح الذي يميز قيمة الحقل عند الإرسال.

<input type="text" name="username">

السمة size

تُستخدم لتحديد عدد الأحرف الظاهرة داخل حقل الإدخال.

<input type="text" size="20">

السمة min

تُستخدم لتحديد أقل قيمة مسموح بها في الحقول الرقمية.

<input type="number" min="1">

السمة max

تُستخدم لتحديد أكبر قيمة مسموح بها.

<input type="number" max="100">

السمة minlength

تُستخدم لتحديد أقل عدد من الأحرف المطلوبة.

<input type="text" minlength="5">

السمة maxlength

تُستخدم لتحديد أكبر عدد من الأحرف المسموح بها.

<input type="text" maxlength="20">

السمة required

أدوات HTML

مدقق HTML

يُستخدم مدقق HTML للتحقق من صحة بنية الكود واكتشاف الأخطاء والمشكلات الموجودة داخل الصفحة.

مدقق DOM

يساعد مدقق DOM على فحص العناصر الموجودة داخل الصفحة ومعرفة العلاقات بينها أثناء تشغيل الموقع.

أدوات المطورين

توفر المتصفحات الحديثة أدوات مطورين قوية تساعد في فحص العناصر وتصحيح الأخطاء وتحليل أداء الموقع.

مقدمة في إمكانية الوصول Accessibility

إمكانية الوصول هي مجموعة من الممارسات التي تهدف إلى جعل المواقع والتطبيقات قابلة للاستخدام من قبل جميع الأشخاص بما في ذلك ذوو الإعاقات المختلفة.

إرشادات WCAG

تُعد إرشادات WCAG من أشهر المعايير المستخدمة لتحسين إمكانية الوصول. وترتكز على أربعة مبادئ رئيسية تعرف باسم POUR.

P = Perceivable O = Operable U = Understandable R = Robust

قابلية الإدراك Perceivable

يجب أن يكون المحتوى قابلاً للإدراك من قبل المستخدمين سواء من خلال الرؤية أو السمع أو وسائل مساعدة أخرى.

قابلية التشغيل Operable

يجب أن تكون جميع عناصر الموقع قابلة للاستخدام بواسطة لوحة المفاتيح أو وسائل الإدخال الأخرى.

قابلية الفهم Understandable

يجب أن يكون المحتوى وسلوك الموقع واضحين وسهلين للفهم.

المتانة Robust

يجب أن يعمل الموقع بشكل جيد مع المتصفحات والتقنيات المساعدة المختلفة.

التقنيات المساعدة

قارئات الشاشة

هي برامج تقوم بقراءة محتوى الشاشة صوتيًا لمساعدة المكفوفين وضعاف البصر على استخدام الإنترنت.

لوحات مفاتيح برايل

تساعد المستخدمين ذوي الإعاقات البصرية على التفاعل مع المحتوى باستخدام نظام برايل.

مكبرات الشاشة

تُستخدم لتكبير أجزاء الشاشة لتسهيل قراءة المحتوى.

أجهزة التأشير البديلة

تشمل عصي التحكم ولوحات اللمس وكرات التتبع لمساعدة الأشخاص ذوي الإعاقات الحركية.

التعرف على الصوت

يسمح للمستخدمين بالتحكم في الحاسوب باستخدام الأوامر الصوتية.

أدوات تدقيق إمكانية الوصول

هناك العديد من الأدوات المستخدمة لاكتشاف مشاكل إمكانية الوصول داخل المواقع الإلكترونية.

أدوات شائعة

Google Lighthouse Wave IBM Equal Accessibility Checker axe DevTools
أفضل الممارسات في إمكانية الوصول

تساعد أفضل ممارسات إمكانية الوصول على جعل مواقع الويب قابلة للاستخدام من قبل جميع الأشخاص، بما في ذلك المستخدمون الذين يعتمدون على التقنيات المساعدة مثل قارئات الشاشة ولوحات المفاتيح.

استخدام بنية عناوين صحيحة

يجب استخدام عناصر العناوين بالترتيب الصحيح من h1 إلى h6 لإنشاء هيكل منطقي للمحتوى. يساعد ذلك التقنيات المساعدة ومحركات البحث على فهم الصفحة بشكل أفضل.

<h1>Main Title</h1> <h2>Section Title</h2> <h3>Subsection Title</h3>

إمكانية الوصول والجداول

عند إنشاء الجداول يجب استخدام العنصر th لخلايا العناوين والعنصر td لخلايا البيانات. يساعد ذلك قارئات الشاشة على تفسير بنية الجدول بشكل صحيح.

<tr> <th>Name</th> <th>Grade</th> </tr> <tr> <td>Osama</td> <td>95</td> </tr>

أهمية ربط label مع input

يجب ربط عناصر label مع حقول الإدخال input حتى تتمكن التقنيات المساعدة من معرفة الغرض من كل حقل.

<label for="email"> Email Address </label> <input type="email" id="email">

أهمية النص البديل alt

تُستخدم السمة alt لتوفير وصف نصي للصور. ويُعد ذلك ضروريًا للمستخدمين الذين يعتمدون على قارئات الشاشة أو في حالة فشل تحميل الصورة.

<img src="logo.png" alt="Company Logo">

أهمية نص الرابط الوصفي

يجب أن يوضح نص الرابط الغرض منه بشكل مباشر بدلًا من استخدام عبارات عامة مثل Click Here.

<a href="courses.html"> Browse Programming Courses </a>

إمكانية الوصول للمحتوى الصوتي والمرئي

يجب توفير ترجمات نصية للمحتوى الصوتي والفيديوهات لمساعدة الأشخاص ذوي الإعاقات السمعية. كما يُفضل إضافة أوصاف صوتية للمحتوى المرئي لمساعدة ذوي الإعاقات البصرية.

خصائص التنقل باستخدام لوحة المفاتيح

السمة tabindex

تُستخدم السمة tabindex لجعل العناصر قابلة للتركيز باستخدام لوحة المفاتيح وتحديد ترتيب التنقل بينها.

يُفضل استخدام القيمة 0 أو -1 وعدم استخدام قيم أكبر من 0 إلا في حالات خاصة.

<button tabindex="0"> Submit </button>

السمة accesskey

تُستخدم السمة accesskey لتحديد اختصار لوحة مفاتيح يسمح بالوصول السريع إلى عنصر معين.

<button accesskey="s"> Save </button>
WAI-ARIA والأدوار والخصائص

WAI-ARIA اختصار لـ Web Accessibility Initiative - Accessible Rich Internet Applications. وهي مجموعة من السمات التي تُضاف إلى عناصر HTML لتحسين إمكانية الوصول وتوفير معلومات إضافية للتقنيات المساعدة.

أدوار ARIA

تُستخدم الأدوار Roles لتوضيح وظيفة العنصر داخل الصفحة عندما لا يكون معناها واضحًا من HTML وحدها.

<div role="alert"> Error Message </div> <div role="tab"> Profile </div> <div role="menu"> Navigation Menu </div>

السمة aria-label

تُستخدم لإعطاء العنصر اسمًا يمكن للتقنيات المساعدة قراءته حتى إذا لم يكن هناك نص مرئي داخل العنصر.

<button aria-label="Close Menu"> ✖ </button>

السمة aria-labelledby

تُستخدم لربط العنصر بنص موجود بالفعل داخل الصفحة ليصبح هذا النص هو الاسم الذي تقرأه التقنيات المساعدة.

<h2 id="form-title"> Contact Form </h2> <form aria-labelledby="form-title"> </form>

السمة aria-hidden

تُستخدم لإخفاء العناصر من قارئات الشاشة عندما لا تحتوي على معلومات مفيدة للمستخدم.

<span aria-hidden="true"> ★ </span>

السمة aria-describedby

تُستخدم لربط عنصر ما بوصف إضافي يساعد المستخدم على فهم الغرض منه أو طريقة استخدامه.

<p id="password-help"> Password must be at least 8 characters. </p> <input type="password" aria-describedby="password-help">

متى نستخدم ARIA؟

يجب الاعتماد أولًا على عناصر HTML الدلالية مثل header و nav و section و article. ويتم استخدام ARIA فقط عندما لا توفر عناصر HTML المعلومات الكافية للتقنيات المساعدة.

Use Semantic HTML First Use ARIA Only When Needed
ملخص HTML

HTML هي لغة بناء صفحات الويب والمسؤولة عن إنشاء هيكل الموقع وتنظيم محتواه.

تشمل HTML العديد من المفاهيم المهمة مثل العناصر الأساسية، النماذج، الجداول، الوسائط المتعددة، الروابط، العناصر الدلالية، وإمكانية الوصول.

كلما تم استخدام عناصر HTML الصحيحة بطريقة منظمة، أصبح الموقع أكثر احترافية وأسهل في الصيانة والتطوير وتحسين محركات البحث.

أساسيات CSS

CSS أو Cascading Style Sheets هي اللغة المسؤولة عن تنسيق صفحات الويب وإضافة الألوان والخلفيات والتخطيطات والتأثيرات البصرية لعناصر HTML.

تعمل CSS جنبًا إلى جنب مع HTML حيث تتولى HTML إنشاء هيكل الصفحة بينما تتولى CSS تحسين مظهرها وتصميمها.

ما هي CSS؟

تُستخدم CSS للتحكم في مظهر العناصر داخل صفحة الويب مثل الألوان والخطوط والمسافات والأحجام والتخطيطات.

p { color: blue; }

البنية الأساسية لقاعدة CSS

تتكون قاعدة CSS من محدد Selector وكتلة تعريف Declaration Block تحتوي على الخصائص والقيم.

p { color: red; font-size: 18px; }

في المثال السابق:

p => Selector color => Property red => Value
عنصر Viewport

يُستخدم العنصر meta viewport للتحكم في كيفية عرض الصفحة على الأجهزة المختلفة مثل الهواتف والأجهزة اللوحية.

إضافة Viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0">

بدون هذا العنصر قد لا تظهر الصفحة بشكل متجاوب على الشاشات الصغيرة.

أنماط المتصفح الافتراضية

تقوم المتصفحات بإضافة أنماط افتراضية لبعض العناصر مثل الهوامش والحشوات والعناوين.

لذلك يستخدم المطورون غالبًا Reset CSS أو Normalization للحصول على نقطة بداية موحدة.

* { margin: 0; padding: 0; box-sizing: border-box; }
أنواع CSS

CSS المضمنة Inline CSS

تُكتب داخل العنصر مباشرة باستخدام السمة style.

<p style="color:red;"> Hello </p>

لا يُنصح باستخدامها كثيرًا لأنها تخالف مبدأ فصل المحتوى عن التصميم.

CSS الداخلية Internal CSS

تُكتب داخل عنصر style في قسم head.

<style> p { color: blue; } </style>

CSS الخارجية External CSS

تُكتب داخل ملف منفصل بامتداد css ويتم ربطه بملف HTML.

<link rel="stylesheet" href="style.css">

تُعتبر الطريقة الأفضل والأكثر استخدامًا في المشاريع الحقيقية.

خصائص العرض والارتفاع

الخاصية width

تُستخدم لتحديد عرض العنصر.

div { width: 300px; }

الخاصية min-width

تُحدد أقل عرض يمكن أن يصل إليه العنصر.

div { min-width: 200px; }

الخاصية max-width

تُحدد أكبر عرض يمكن أن يصل إليه العنصر.

div { max-width: 800px; }

الخاصية height

تُستخدم لتحديد ارتفاع العنصر.

div { height: 300px; }

الخاصية min-height

تُحدد أقل ارتفاع مسموح به للعنصر.

div { min-height: 150px; }

الخاصية max-height

تُحدد أكبر ارتفاع يمكن أن يصل إليه العنصر.

div { max-height: 600px; }
مركبات CSS

تُستخدم المركبات Combinators لتحديد العلاقة بين العناصر داخل الصفحة.

مركب العناصر التابعة Descendant

يستهدف جميع العناصر الموجودة داخل عنصر أب معين.

div p { color: red; }

مركب الأبناء Child

يستهدف الأبناء المباشرين فقط.

div > p { color: blue; }

مركب الأخ المجاور Adjacent Sibling

يستهدف العنصر الذي يأتي مباشرة بعد عنصر آخر.

h1 + p { color: green; }

مركب الأشقاء اللاحقين General Sibling

يستهدف جميع الأشقاء الذين يأتون بعد العنصر المحدد.

h1 ~ p { color: purple; }
أنواع عناصر العرض

تتعامل CSS مع العناصر بطرق مختلفة حسب نوع العرض الخاص بها. وتنقسم العناصر غالبًا إلى عناصر مضمنة Inline وعناصر كتلية Block وعناصر Inline-Block.

العناصر المضمنة Inline Elements

العناصر المضمنة تشغل مساحة مساوية لحجم محتواها فقط ولا تبدأ في سطر جديد. يمكن أن تظهر بجانب عناصر أخرى في نفس السطر.

<span>Text</span> <a href="#">Link</a> <img src="image.jpg">

العناصر الكتلية Block Elements

تشغل العناصر الكتلية العرض الكامل المتاح لها بشكل افتراضي وتبدأ دائمًا في سطر جديد.

<div>Content</div> <p>Paragraph</p> <section>Section</section>

العناصر Inline-Block

تجمع بين خصائص العناصر المضمنة والكتلية. يمكن أن تظهر بجانب العناصر الأخرى مع إمكانية تحديد العرض والارتفاع لها.

.box { display: inline-block; }
الهوامش والحشوات

الخاصية margin

تُستخدم الخاصية margin لإنشاء مساحة خارجية بين العنصر والعناصر المحيطة به.

div { margin: 20px; }

اختصار margin

يمكن استخدام قيمة واحدة أو أكثر للتحكم في جميع الجوانب.

margin: 20px; margin: 10px 20px; margin: 10px 20px 30px; margin: 10px 20px 30px 40px;

ترتيب القيم يكون: Top → Right → Bottom → Left

الخاصية padding

تُستخدم الخاصية padding لإنشاء مساحة داخلية بين المحتوى وحدود العنصر.

div { padding: 20px; }

اختصار padding

padding: 20px; padding: 10px 20px; padding: 10px 20px 30px; padding: 10px 20px 30px 40px;
خصوصية CSS

عندما تتعارض أكثر من قاعدة CSS على نفس العنصر، يستخدم المتصفح مفهوم Specificity لتحديد القاعدة التي سيتم تطبيقها.

خصوصية الأنماط المضمنة

تتمتع الأنماط المضمنة بأعلى أولوية تقريبًا لأنها تُكتب مباشرة داخل العنصر.

<p style="color:red;"> Hello </p>

قيمة الخصوصية:

(1,0,0,0)

خصوصية المحدد العام *

المحدد العام يطابق جميع العناصر ويملك أقل قيمة خصوصية.

* { margin: 0; } (0,0,0,0)

محددات النوع Type Selectors

تستهدف العناصر حسب اسم الوسم.

p { color: blue; } (0,0,0,1)

محددات الفئات Class Selectors

تستهدف العناصر حسب اسم الفئة.

.card { color: red; } (0,0,1,0)

محددات المعرفات ID Selectors

تتمتع بقوة أعلى من محددات الفئات والأنواع.

#hero { color: green; } (0,1,0,0)

ترتيب الأولوية

Inline Style ID Selector Class Selector Type Selector Universal Selector
الكلمة المفتاحية !important

تُستخدم !important لإجبار المتصفح على تطبيق قاعدة معينة حتى لو كانت هناك قواعد أخرى ذات خصوصية أعلى.

p { color: red !important; }

يجب استخدام هذه الكلمة بحذر لأنها قد تجعل ملفات CSS أكثر صعوبة في الصيانة والتطوير.

التوريث في CSS

التوريث هو انتقال بعض الخصائص من العنصر الأب إلى العناصر الأبناء تلقائيًا.

بعض الخصائص مثل اللون والخطوط يتم توريثها افتراضيًا. بينما خصائص أخرى مثل العرض والهوامش لا يتم توريثها.

body { color: blue; }

في المثال السابق سترث العناصر الموجودة داخل body اللون الأزرق ما لم يتم تغييره.

التوازن هو توزيع الوزن البصري للعناصر داخل الصفحة. يمكن تحقيقه بشكل متماثل أو غير متماثل حسب طبيعة التصميم.

المقياس Scale

يشير المقياس إلى مقارنة أحجام العناصر ببعضها البعض. يُستخدم لإبراز العناصر المهمة أو إنشاء تسلسل بصري واضح.

التسلسل الهرمي Hierarchy

الوحدات المطلقة

الوحدات المطلقة هي وحدات ثابتة لا تعتمد على حجم الشاشة أو العنصر الأب.

وحدة px

تُعد أكثر الوحدات استخدامًا في CSS وتمثل البكسل.

div { width: 300px; }

وحدة in

تمثل البوصة الواحدة وتساوي 96 بكسل.

width: 2in;

وحدة cm

تمثل السنتيمتر وتُستخدم غالبًا في الطباعة.

width: 10cm;

وحدة mm

تمثل الملليمتر.

width: 20mm;

وحدة q

تمثل ربع الملليمتر.

width: 40q;

وحدة pc

تمثل البيكا وتساوي سدس البوصة.

width: 4pc;

وحدة pt

تمثل النقطة وتساوي 1/72 من البوصة.

font-size: 12pt;
الوحدات النسبية

الوحدات النسبية تعتمد على حجم عنصر آخر أو على أبعاد نافذة العرض.

النسبة المئوية %

تعتمد على أبعاد العنصر الأب.

width: 50%;

وحدة em

تعتمد على حجم الخط الخاص بالعنصر الحالي أو العنصر الأب.

font-size: 2em;

وحدة rem

تعتمد على حجم الخط الخاص بالعنصر الجذر html.

font-size: 2rem;

وحدة vh

تمثل نسبة من ارتفاع نافذة العرض. كل 1vh يساوي 1% من ارتفاع الشاشة.

height: 100vh;

وحدة vw

تمثل نسبة من عرض نافذة العرض. كل 1vw يساوي 1% من عرض الشاشة.

width: 100vw;
الدالة calc()

تُستخدم الدالة calc() لإجراء عمليات حسابية مباشرة داخل CSS.

تساعد على إنشاء تصميمات مرنة ومتجاوبة دون الحاجة إلى JavaScript.

الجمع

width: calc(100% - 50px);

الطرح

height: calc(100vh - 80px);

الضرب

width: calc(100px * 2);

القسمة

width: calc(200px / 2);

مثال عملي

.container { width: calc(100% - 40px); margin: 20px; }

في المثال السابق سيتم حساب العرض تلقائيًا بناءً على عرض العنصر الأب مع خصم الهوامش المحددة.

الفئات الزائفة Pseudo Classes

الفئات الزائفة هي كلمات مفتاحية خاصة في CSS تُستخدم لاستهداف العناصر بناءً على حالة معينة أو موقعها داخل الصفحة.

الفئة الزائفة :hover

تُستخدم لتطبيق أنماط عند مرور مؤشر الفأرة فوق العنصر.

button:hover { background-color: blue; }

الفئة الزائفة :active

تُستخدم أثناء الضغط على العنصر.

button:active { transform: scale(0.95); }

الفئة الزائفة :focus

تُطبق الأنماط عندما يحصل العنصر على التركيز.

input:focus { border-color: blue; }

الفئة الزائفة :focus-within

تُطبق الأنماط على العنصر الأب عندما يحصل أي عنصر داخله على التركيز.

.form-group:focus-within { background: #eee; }
فئات الإدخال الزائفة

:enabled

تستهدف العناصر المفعلة التي يمكن للمستخدم التفاعل معها.

input:enabled { background: white; }

:disabled

تستهدف العناصر المعطلة.

input:disabled { opacity: 0.5; }

:checked

تستهدف عناصر الاختيار المحددة.

input:checked { accent-color: green; }

:valid

تستهدف الحقول التي تحتوي على بيانات صحيحة.

input:valid { border-color: green; }

:invalid

تستهدف الحقول التي تحتوي على بيانات غير صحيحة.

input:invalid { border-color: red; }

:in-range

تستهدف القيم الموجودة داخل النطاق المحدد.

input:in-range { background: lightgreen; }

:out-of-range

تستهدف القيم الموجودة خارج النطاق المحدد.

input:out-of-range { background: pink; }

:required

تستهدف الحقول المطلوبة.

input:required { border-left: 3px solid red; }

:optional

تستهدف الحقول غير المطلوبة.

input:optional { border-left: 3px solid gray; }

:autofill

تستهدف الحقول التي ملأها المتصفح تلقائيًا.

input:autofill { background: lightyellow; }
فئات الموقع الزائفة

:any-link

تستهدف جميع الروابط سواء تمت زيارتها أم لا.

:any-link { color: blue; }

:link

تستهدف الروابط غير المزارة.

a:link { color: blue; }

:visited

تستهدف الروابط التي تمت زيارتها.

a:visited { color: purple; }

:local-link

تستهدف الروابط التي تشير إلى نفس المستند أو الموقع.

:target

تستهدف العنصر المشار إليه في عنوان URL.

#about:target { background: yellow; }
الفئات الزائفة ذات البنية الشجرية

:root

تمثل العنصر الجذر للمستند وغالبًا ما تكون html.

:root { --main-color: blue; }

:empty

تستهدف العناصر الفارغة التي لا تحتوي على محتوى.

div:empty { display: none; }

:first-child

تستهدف أول عنصر داخل العنصر الأب.

li:first-child { color: red; }

:last-child

تستهدف آخر عنصر داخل العنصر الأب.

li:last-child { color: blue; }

:only-child

تستهدف العنصر الوحيد داخل العنصر الأب.

p:only-child { font-weight: bold; }

:nth-child(n)

تستهدف عنصرًا حسب ترتيبه.

li:nth-child(2) { color: green; }

:nth-last-child(n)

تعمل مثل nth-child ولكن تبدأ العد من النهاية.

li:nth-last-child(2) { color: orange; }

:first-of-type

تستهدف أول عنصر من نوع معين.

p:first-of-type { color: red; }

:last-of-type

تستهدف آخر عنصر من نوع معين.

p:last-of-type { color: blue; }

:nth-of-type(n)

تستهدف عنصرًا من نفس النوع بناءً على ترتيبه.

p:nth-of-type(3) { color: green; }

:only-of-type

تستهدف العنصر إذا كان الوحيد من نوعه داخل الأب.

p:only-of-type { color: purple; }
الفئات الزائفة الوظيفية

:is()

تسمح بتجميع عدة محددات داخل محدد واحد.

:is(h1, h2, h3) { color: blue; }

:where()

تشبه is لكنها تمتلك خصوصية صفرية.

:where(h1, h2, h3) { margin-bottom: 20px; }

:has()

تُستخدم لاستهداف عنصر يحتوي على عنصر آخر محدد. وتُعرف أحيانًا باسم Parent Selector.

.card:has(img) { border: 2px solid blue; }
العناصر الزائفة Pseudo Elements

::before

يُستخدم لإضافة محتوى قبل العنصر.

h2::before { content: "★ "; }

::after

يُستخدم لإضافة محتوى بعد العنصر.

h2::after { content: " ✔"; }

::first-letter

يستهدف أول حرف داخل العنصر.

p::first-letter { font-size: 2rem; }

::marker

يُستخدم لتنسيق علامات عناصر القوائم.

li::marker { color: red; }
طرق التعامل مع الألوان في CSS

توفر CSS عدة طرق مختلفة لتمثيل الألوان وتطبيقها على العناصر. يمكن للمطور اختيار الطريقة المناسبة حسب احتياجات المشروع وسهولة القراءة والصيانة.

الألوان المسماة

الألوان المسماة هي أسماء ألوان معرفة مسبقًا وتدعمها جميع المتصفحات الحديثة.

color: blue; color: darkred; color: lightgreen;

الدالة rgb()

تعتمد هذه الدالة على نموذج RGB الذي يتكون من الألوان الأساسية للضوء: الأحمر والأخضر والأزرق.

color: rgb(255, 0, 0); color: rgb(0, 255, 0); color: rgb(0, 0, 255);

الدالة rgba()

تشبه rgb() ولكنها تضيف قيمة Alpha للتحكم في مستوى الشفافية.

color: rgba(255, 0, 0, 0.5);

الدالة hsl()

تعتمد على ثلاثة مكونات: Hue و Saturation و Lightness.

color: hsl(240, 100%, 50%);

الدالة hsla()

تشبه hsl() ولكنها تسمح بإضافة قيمة Alpha للتحكم في الشفافية.

color: hsla(240, 100%, 50%, 0.5);

الألوان السداسية Hexadecimal

تُستخدم القيم السداسية لتمثيل الألوان باستخدام ستة رموز تتكون من الأرقام والحروف.

color: #ff0000; color: #00ff00; color: #0000ff;
خاصية box-shadow

تُستخدم خاصية box-shadow لإضافة ظل حول العناصر مما يساعد على إبرازها وإضافة عمق للتصميم.

بنية الخاصية

box-shadow: offset-x offset-y blur-radius spread-radius color;

طرق التعامل مع الألوان في CSS

توفر CSS عدة طرق مختلفة لتمثيل الألوان وتطبيقها على العناصر. يمكن للمطور اختيار الطريقة المناسبة حسب احتياجات المشروع وسهولة القراءة والصيانة.

الألوان المسماة

الألوان المسماة هي أسماء ألوان معرفة مسبقًا وتدعمها جميع المتصفحات الحديثة.

color: blue; color: darkred; color: lightgreen;

الدالة rgb()

تعتمد هذه الدالة على نموذج RGB الذي يتكون من الألوان الأساسية للضوء: الأحمر والأخضر والأزرق.

color: rgb(255, 0, 0); color: rgb(0, 255, 0); color: rgb(0, 0, 255);

الدالة rgba()

تشبه rgb() ولكنها تضيف قيمة Alpha للتحكم في مستوى الشفافية.

color: rgba(255, 0, 0, 0.5);

الدالة hsl()

تعتمد على ثلاثة مكونات: Hue و Saturation و Lightness.

color: hsl(240, 100%, 50%);

الدالة hsla()

تشبه hsl() ولكنها تسمح بإضافة قيمة Alpha للتحكم في الشفافية.

color: hsla(240, 100%, 50%, 0.5);

الألوان السداسية Hexadecimal

تُستخدم القيم السداسية لتمثيل الألوان باستخدام ستة رموز تتكون من الأرقام والحروف.

color: #ff0000; color: #00ff00; color: #0000ff;

input { appearance: none; }
خاصية Overflow في CSS

تشير خاصية Overflow إلى كيفية تعامل العنصر مع المحتوى الذي يتجاوز أبعاده المحددة. عندما يكون المحتوى أكبر من مساحة العنصر فإنه يُعتبر محتوى فائضًا.

الخاصية overflow-x

تتحكم هذه الخاصية في كيفية التعامل مع الفائض الأفقي للمحتوى.

.container { overflow-x: auto; }

الخاصية overflow-y

تتحكم هذه الخاصية في كيفية التعامل مع الفائض الرأسي للمحتوى.

.container { overflow-y: scroll; }
خاصية Transform

تُستخدم خاصية Transform لتطبيق التحويلات المختلفة على العناصر مثل النقل والتكبير والتدوير والتحريف.

الدالة translate()

تُستخدم لنقل العنصر من موضعه الأصلي إلى موضع جديد.

.box { transform: translate(50px, 20px); }

الدالة scale()

تُستخدم لتكبير أو تصغير حجم العنصر.

.box { transform: scale(1.5); }

التحويلات وإمكانية الوصول

إذا تم استخدام التحويلات لإخفاء المحتوى أو إظهاره فيجب التأكد من أن المحتوى ما زال متاحًا للتقنيات المساعدة ولوحة المفاتيح.

display: none; visibility: hidden;
نموذج الصندوق Box Model

كل عنصر في CSS يُعتبر صندوقًا يتكون من عدة أجزاء مترابطة. يساعد فهم نموذج الصندوق على التحكم الدقيق في أبعاد العناصر والمسافات بينها.

منطقة المحتوى Content Area

هي المنطقة التي تحتوي على المحتوى الفعلي للعنصر مثل النصوص والصور.

الحشو Padding

هو المسافة بين المحتوى وحدود العنصر.

padding: 20px;

الحدود Border

تمثل الإطار المحيط بالعنصر.

border: 2px solid black;

الهامش Margin

هو المسافة الخارجية بين العنصر والعناصر المحيطة به.

margin: 20px;

مكونات Box Model

Content Padding Border Margin
انهيار الهوامش Margin Collapse

يحدث انهيار الهوامش عندما تتداخل الهوامش الرأسية لعناصر متجاورة. في هذه الحالة يتم استخدام أكبر قيمة هامش بدلًا من جمع القيمتين.

div { margin-bottom: 30px; } section { margin-top: 20px; }

في المثال السابق ستكون المسافة النهائية 30px وليس 50px.

خاصية Box Sizing

تُستخدم خاصية box-sizing لتحديد طريقة حساب العرض والارتفاع النهائي للعنصر.

القيمة content-box

وهي القيمة الافتراضية. يتم حساب العرض والارتفاع للمحتوى فقط دون الحشو والحدود.

box-sizing: content-box;

القيمة border-box

تشمل العرض والارتفاع المحتوى والحشو والحدود. وهي الأكثر استخدامًا في المشاريع الحديثة.

box-sizing: border-box;

إعادة ضبط شائعة

* { box-sizing: border-box; }
إعادة ضبط CSS

تُستخدم إعادة الضبط لإزالة التنسيقات الافتراضية التي تطبقها المتصفحات على العناصر.

أهمية CSS Reset

تساعد على توحيد مظهر الموقع عبر مختلف المتصفحات.

أشهر الأدوات

normalize.css sanitize.css

مثال بسيط

* { margin: 0; padding: 0; }
خاصية Filter

تُستخدم خاصية Filter لتطبيق تأثيرات بصرية مختلفة على الصور والعناصر.

الدالة blur()

تُضيف تأثير التمويه على العنصر.

img { filter: blur(5px); }

الدالة brightness()

تتحكم في مستوى سطوع العنصر.

img { filter: brightness(150%); }

الدالة grayscale()

تحول الصورة إلى تدرج رمادي.

img { filter: grayscale(100%); }

الدالة sepia()

تضيف تأثيرًا بنيًا قديمًا للصورة.

img { filter: sepia(100%); }

الدالة hue-rotate()

تُستخدم لتدوير ألوان العنصر حول عجلة الألوان.

img { filter: hue-rotate(90deg); }
مقدمة إلى CSS Flexbox

يُعتبر Flexbox أحد أهم أنظمة التخطيط في CSS. يُستخدم لتنظيم العناصر داخل الحاويات بطريقة مرنة وسهلة سواء أفقيًا أو رأسيًا.

يُعد Flexbox نموذج تخطيط أحادي البعد لأنه يتعامل مع محور واحد في كل مرة.

تفعيل Flexbox

لتحويل عنصر إلى حاوية مرنة يجب استخدام الخاصية display بالقيمة flex.

.container { display: flex; }
نموذج Flex

عند استخدام Flexbox يصبح لدينا محوران رئيسيان: المحور الرئيسي Main Axis والمحور العرضي Cross Axis.

المحور الرئيسي

هو المحور الذي يتم ترتيب العناصر عليه افتراضيًا.

المحور العرضي

هو المحور العمودي على المحور الرئيسي.

الخاصية flex-direction

تُستخدم هذه الخاصية لتحديد اتجاه المحور الرئيسي وطريقة ترتيب العناصر داخله.

القيمة row

وهي القيمة الافتراضية، حيث يتم ترتيب العناصر أفقيًا.

.container { flex-direction: row; }

القيمة row-reverse

تعكس ترتيب العناصر أفقيًا.

.container { flex-direction: row-reverse; }

القيمة column

ترتب العناصر رأسيًا من الأعلى إلى الأسفل.

.container { flex-direction: column; }

القيمة column-reverse

تعكس ترتيب العناصر رأسيًا.

.container { flex-direction: column-reverse; }
الخاصية flex-wrap

تتحكم هذه الخاصية في إمكانية انتقال العناصر إلى سطر جديد عندما لا تتسع المساحة المتاحة.

القيمة nowrap

وهي القيمة الافتراضية. لن تنتقل العناصر إلى سطر جديد حتى لو تجاوزت مساحة الحاوية.

.container { flex-wrap: nowrap; }

القيمة wrap

تسمح للعناصر بالانتقال إلى سطر جديد عند الحاجة.

.container { flex-wrap: wrap; }

القيمة wrap-reverse

تشبه wrap ولكن بترتيب معكوس.

.container { flex-wrap: wrap-reverse; }
الخاصية flex-flow

تُعتبر اختصارًا للخاصيتين flex-direction و flex-wrap.

.container { flex-flow: row wrap; }

يعادل المثال السابق:

.container { flex-direction: row; flex-wrap: wrap; }
الخاصية justify-content

تُستخدم لمحاذاة العناصر على طول المحور الرئيسي.

القيمة flex-start

تُحاذي العناصر إلى بداية المحور الرئيسي.

justify-content: flex-start;

القيمة flex-end

تُحاذي العناصر إلى نهاية المحور الرئيسي.

justify-content: flex-end;

القيمة center

تُوسّط العناصر داخل الحاوية.

justify-content: center;

القيمة space-between

توزع العناصر بالتساوي مع ترك مسافات بين العناصر فقط.

justify-content: space-between;

القيمة space-around

تضيف مسافات حول جميع العناصر.

justify-content: space-around;

القيمة space-evenly

توزع العناصر بحيث تكون جميع المسافات متساوية تمامًا.

justify-content: space-evenly;
الخاصية align-items

تُستخدم لمحاذاة العناصر على طول المحور العرضي.

القيمة flex-start

تُحاذي العناصر إلى بداية المحور العرضي.

align-items: flex-start;

القيمة center

تُوسّط العناصر على المحور العرضي.

align-items: center;

القيمة stretch

تُمدد العناصر لتملأ المساحة المتاحة على المحور العرضي.

align-items: stretch;

مثال كامل

.container { display: flex; flex-direction: row; flex-wrap: wrap; justify-content: center; align-items: center; }

في المثال السابق سيتم ترتيب العناصر داخل الحاوية باستخدام Flexbox مع توسيطها أفقيًا ورأسيًا.

مقدمة في فن الطباعة

الطباعة هي فن اختيار الخطوط والتنسيق المناسبين لجعل النص جذابًا بصريًا وسهل القراءة. يشير مصطلح الخط إلى كيفية تصميم الأحرف الفردية وترتيبها.

تعريف Typeface

الـ Typeface هو التصميم العام وأسلوب مجموعة من الأحرف والأرقام والرموز. يمكن اعتباره المخطط الأساسي الذي تُبنى عليه الخطوط المختلفة.

تعريف Font

الـ Font هو شكل محدد من أشكال Typeface وله خصائص معينة مثل الحجم والوزن والعرض والنمط.

أنواع الخطوط

الخطوط ذات الزوائد Serif

تتميز بوجود خطوط صغيرة أو زوائد في نهايات الأحرف. تُستخدم غالبًا في الكتب والمطبوعات التقليدية.

الخطوط غير المزخرفة Sans-Serif

لا تحتوي على الزوائد الموجودة في خطوط Serif. تمتاز بسهولة القراءة على الشاشات الحديثة.

Helvetica Arial Roboto
خصائص الخطوط

وزن الخط Font Weight

يشير إلى سمك الأحرف داخل الخط.

Light Regular Bold Black

نمط الخط Font Style

يشير إلى شكل الخط مثل الخط العادي أو المائل.

normal italic oblique

عرض الخط Font Width

يشير إلى المساحة الأفقية التي تشغلها الأحرف. قد تكون الأحرف مضغوطة أو موسعة.

تشريح الخطوط

خط الأساس Baseline

هو الخط الوهمي الذي ترتكز عليه معظم الأحرف.

ارتفاع الأحرف الكبيرة Cap Height

هو ارتفاع الأحرف الكبيرة من خط الأساس إلى أعلى الحرف.

ارتفاع الحرف X

يمثل متوسط ارتفاع الأحرف الصغيرة باستثناء الأحرف الصاعدة والهابطة.

الأحرف الصاعدة Ascenders

هي أجزاء الأحرف التي تمتد فوق ارتفاع x.

h b d f

الأحرف الهابطة Descenders

هي أجزاء الأحرف التي تمتد أسفل خط الأساس.

y g p q
المسافات في الطباعة

Kerning

هو تعديل المسافة بين زوج محدد من الأحرف لتحسين المظهر البصري وسهولة القراءة.

مثال شائع هو تقليل المسافة بين الحرفين A و V.

Tracking

هو التحكم في المسافة بين جميع الأحرف داخل كلمة أو فقرة كاملة.

Line Height

هو المسافة الرأسية بين الأسطر. ويتم قياسها من خط الأساس لسطر إلى خط الأساس للسطر التالي.

أفضل الممارسات في الطباعة

يجب اختيار خطوط واضحة وسهلة القراءة خاصة للنصوص الأساسية داخل الموقع.

كلما كان الخط أكثر وضوحًا زادت قابلية قراءة المحتوى وتحسنت تجربة المستخدم.

يفضل استخدام خطين أو ثلاثة خطوط فقط داخل المشروع للحفاظ على التناسق البصري.

الإفراط في استخدام الخطوط المختلفة قد يسبب تشتيتًا للمستخدم ويؤثر على هوية التصميم.

يمكن استخدام أحجام الخطوط المختلفة لإنشاء تسلسل هرمي بصري واضح بين العناوين الرئيسية والعناوين الفرعية والفقرات.

الخاصية font-family

تُستخدم الخاصية font-family لتحديد الخطوط المستخدمة داخل العنصر.

يمكن تحديد أكثر من خط كخيارات احتياطية في حالة عدم توفر الخط الأساسي.

body { font-family: Arial, Helvetica, sans-serif; }

الخطوط الآمنة للويب

إذا لم يجد المتصفح الخط المطلوب فسيستخدم الخط الاحتياطي أو الخط الافتراضي للنظام.

قواعد At Rules

قواعد At Rules هي تعليمات خاصة يتم تمريرها إلى المتصفح للتحكم في سلوك ورقة الأنماط.

يمكن استخدامها للتعامل مع الخطوط المخصصة واستعلامات الوسائط والرسوم المتحركة والعديد من الميزات الأخرى.

@media @font-face @keyframes
القاعدة @font-face

تُستخدم القاعدة @font-face لتعريف خطوط مخصصة داخل المشروع.

تسمح هذه القاعدة باستخدام ملفات خطوط خارجية حتى لو لم تكن مثبتة على جهاز المستخدم.

الخاصية src

تُستخدم لتحديد مسار ملف الخط الذي سيتم تحميله.

@font-face { font-family: "MyFont"; src: url("fonts/my-font.woff2") format("woff2"); }

لكي تكون القاعدة @font-face صحيحة يجب توفير الخاصية src.

شجرة إمكانية الوصول

شجرة إمكانية الوصول هي بنية تستخدمها التقنيات المساعدة مثل قارئات الشاشة لفهم محتوى الصفحة والتفاعل معه.

تعتمد برامج قراءة الشاشة على هذه الشجرة بدلًا من الاعتماد على العرض المرئي للصفحة.

الدالة max()

تُرجع الدالة max() أكبر قيمة من مجموعة من القيم المفصولة بفواصل.

img { width: max(250px, 25vw); }

إذا كانت القيمة الأولى أكبر فسيتم استخدامها، وإذا كانت القيمة الثانية أكبر فسيتم استخدامها بدلًا منها.

تُستخدم هذه الدالة بشكل شائع في التصميمات المتجاوبة.

أفضل الممارسات في استخدام CSS وإمكانية الوصول

تُعد إمكانية الوصول Accessibility جزءًا مهمًا من تطوير الويب، حيث تساعد جميع المستخدمين بمن فيهم مستخدمو التقنيات المساعدة على الوصول إلى المحتوى والتفاعل معه بسهولة.

استخدام display: none

عند استخدام الخاصية display بالقيمة none يتم إخفاء العنصر بالكامل من الصفحة.

لن يظهر العنصر بصريًا كما أنه سيُزال من شجرة إمكانية الوصول.

لن تتمكن قارئات الشاشة أو التقنيات المساعدة الأخرى من الوصول إلى هذا المحتوى.

.hidden { display: none; }

يجب استخدام هذه الطريقة فقط عندما يكون الهدف هو إزالة المحتوى بالكامل من العرض وإمكانية الوصول.

استخدام visibility: hidden

تُخفي هذه الخاصية العنصر بصريًا مع احتفاظه بمساحته داخل التخطيط.

يظل العنصر موجودًا داخل المستند ولكنه لا يكون مرئيًا للمستخدم.

كما يتم إزالته من شجرة إمكانية الوصول ولن تتمكن قارئات الشاشة من قراءته.

.hidden { visibility: hidden; }
الفئة .sr-only

تُستخدم هذه التقنية لإخفاء المحتوى بصريًا مع إبقائه متاحًا لقارئات الشاشة.

تُعتبر من أشهر تقنيات إمكانية الوصول المستخدمة في النماذج والأزرار والأيقونات.

.sr-only { position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px; overflow: hidden; clip: rect(0, 0, 0, 0); white-space: nowrap; border: 0; }

الميزة prefers-reduced-motion

تُستخدم هذه الميزة لاكتشاف ما إذا كان المستخدم يفضل تقليل الحركة والرسوم المتحركة على جهازه.

يُعتبر استخدامها من أفضل ممارسات إمكانية الوصول.

@media (prefers-reduced-motion: no-preference) { * { scroll-behavior: smooth; } }

في المثال السابق يتم تفعيل التمرير السلس فقط إذا لم يكن المستخدم قد طلب تقليل الحركة.

أنظمة التموضع في CSS

تحدد خاصية position الطريقة التي يتم بها وضع العنصر داخل الصفحة.

توجد عدة أنواع من التموضع لكل منها استخدامات مختلفة.

التموضع الثابت Static

يُعد التموضع الثابت القيمة الافتراضية لجميع العناصر.

تظهر العناصر داخل التدفق الطبيعي للمستند من الأعلى إلى الأسفل ومن اليسار إلى اليمين.

position: static;

في هذا النوع لا تعمل الخصائص top و right و bottom و left.

التموضع النسبي Relative

يبقى العنصر داخل التدفق الطبيعي للمستند ولكن يمكن تحريكه بالنسبة إلى موقعه الأصلي.

يتم استخدام الخصائص top و right و bottom و left لتحديد مقدار الإزاحة.

.box { position: relative; top: 20px; left: 30px; }

يظل مكان العنصر الأصلي محفوظًا داخل التخطيط.

التموضع المطلق Absolute

يقوم التموضع المطلق بإخراج العنصر من التدفق الطبيعي للمستند.

يتم تحديد موقع العنصر بالنسبة لأقرب عنصر أب يمتلك position غير static.

.box { position: absolute; top: 0; right: 0; }

إذا لم يوجد عنصر أب متموضع فسيتم التموضع بالنسبة إلى الصفحة نفسها.

التموضع الثابت Fixed

عند استخدام position: fixed يتم إخراج العنصر من التدفق الطبيعي للمستند.

يتم تموضع العنصر بالنسبة إلى نافذة العرض Viewport وليس بالنسبة للعناصر الأخرى.

يبقى العنصر في نفس المكان حتى عند قيام المستخدم بالتمرير داخل الصفحة.

header { position: fixed; top: 0; width: 100%; }

التموضع اللاصق Sticky

يُعتبر التموضع اللاصق مزيجًا بين التموضع النسبي والتموضع الثابت.

في البداية يتصرف العنصر كعنصر نسبي داخل الصفحة.

وعندما يصل المستخدم إلى نقطة معينة أثناء التمرير يصبح العنصر مثبتًا داخل نافذة العرض.

الخاصية z-index

تُستخدم الخاصية z-index للتحكم في ترتيب تراكب العناصر فوق بعضها البعض.

كلما زادت قيمة z-index ظهر العنصر فوق العناصر ذات القيم الأقل.

.box-one { position: relative; z-index: 10; } .box-two { position: relative; z-index: 1; }

في المثال السابق سيظهر العنصر الأول فوق العنصر الثاني.

ملاحظات مهمة

تعمل الخاصية z-index فقط على العناصر المتموضعة Positioned Elements.

position: relative absolute fixed sticky

لا يكون لها تأثير عادةً على العناصر ذات التموضع static.

مقارنة بين أنواع التموضع

Static

التموضع الافتراضي داخل التدفق الطبيعي للمستند.

Relative

يبقى داخل التدفق الطبيعي مع إمكانية تحريكه من موضعه الأصلي.

Absolute

يخرج من التدفق الطبيعي ويتم تموضعه بالنسبة لأقرب عنصر أب متموضع.

Fixed

يبقى ثابتًا بالنسبة لنافذة العرض حتى أثناء التمرير.

Sticky

يتحول من عنصر نسبي إلى عنصر ثابت عند الوصول إلى نقطة معينة أثناء التمرير.

تصميم مواقع الويب المتجاوبة

التصميم المتجاوب Responsive Design هو أسلوب تطوير يهدف إلى جعل مواقع الويب قادرة على التكيف مع مختلف أحجام الشاشات والأجهزة.

يعتمد هذا المفهوم على مبدأ التكيف بحيث يتغير التخطيط والمحتوى تلقائيًا وفقًا لحجم الشاشة وإمكانات الجهاز المستخدم.

أساسيات CSS Animations

تُستخدم الرسوم المتحركة في CSS لإنشاء تأثيرات بصرية ديناميكية دون الحاجة إلى JavaScript.

القاعدة @keyframes

تُستخدم لتحديد مراحل الرسوم المتحركة المختلفة.

@keyframes move { from { transform: translateX(0); } to { transform: translateX(200px); } }
الخاصية animation

تُعتبر اختصارًا لجميع خصائص الرسوم المتحركة.

.box { animation: move 2s ease-in-out infinite; }
خصائص Animation
animation-name /* يحدد اسم الحركة المرتبطة بـ @keyframes */ animation-duration /* يحدد مدة تنفيذ الحركة. مثال: 2s = ثانيتان ، 500ms = نصف ثانية */ animation-timing-function /* يحدد أسلوب الحركة وسرعتها أثناء التنفيذ. أشهر القيم: linear = سرعة ثابتة ease = بطيئة ثم سريعة ثم بطيئة ease-in = تبدأ ببطء ease-out = تنتهي ببطء ease-in-out = تبدأ وتنتهي ببطء */ animation-delay /* يحدد مدة الانتظار قبل بدء الحركة. مثال: 3s يعني انتظر 3 ثوانٍ ثم ابدأ */ animation-iteration-count /* يحدد عدد مرات تكرار الحركة. 1 = مرة واحدة 3 = ثلاث مرات infinite = تكرار لا نهائي */ animation-direction /* يحدد اتجاه الحركة. normal = الاتجاه الطبيعي reverse = عكس الحركة alternate = ذهاب وعودة alternate-reverse = عودة وذهاب */ animation-fill-mode /* يحدد شكل العنصر قبل أو بعد انتهاء الحركة. none = الوضع الافتراضي forwards = الاحتفاظ بالشكل النهائي backwards = تطبيق أول حالة قبل البداية both = يجمع بين forwards و backwards */ animation-play-state /* التحكم في تشغيل أو إيقاف الحركة. running = تعمل paused = متوقفة مؤقتاً */
